# 素材管理与关联使用说明

素材管理现在围绕 Collection、Entity、标签和知识库治理展开。

## 基础素材库排序

基础素材库模块中的卡片顺序已调整为：

1. Entity DB
2. 标题库
3. 关键词库
4. 图片库
5. Case DB
6. 作者库

业务容器 Collection 不再放在基础素材库中，只保留顶部菜单入口。

## Entity 关联素材

素材可以与 Entity 建立关联，用于提升任务生成时的检索准确性。

建议关联：

- 产品手册关联产品型号 Entity。
- FAQ 关联对应产品线或产品型号 Entity。
- 图片关联具体产品或应用场景 Entity。
- 关键词关联产品型号、产品线或行业 Entity。
- Case 关联涉及的产品、材料、行业或应用场景 Entity。

## 多选与取消关联

素材关联 Entity 的选择器应支持多选和取消关联。

如果某个素材确实只属于一个 Entity，可以只选一个；如果它同时解释多个对象，可以选择多个。

知识库关联 Entity 时支持更细的关系设置：

1. 先在搜索多选框中选择 Entity。
2. 每个已选 Entity 会显示为 `Entity 名称 - 关系`。
3. 可分别设置主资料、辅助参考、应用参考、竞品参考或故障排查参考。
4. 保存后这些关系会参与 RAG 检索和生成上下文判断。

Entity 编辑页关联知识库时使用同一套样式，只是显示为 `知识库名称 - 关系`。

关键词库、图片库和图片素材仍然只保存普通 Entity 关联，不需要为每个 Entity 设置知识关系。

## 标签选择器

标签选择器采用搜索下拉方式：

- 输入为空时显示可选标签。
- 输入文字后实时过滤。
- 点击标签即选择。
- 已选标签以标签云形式显示。
- 删除标签后即时保存。

所有相关输入框已统一去除内部双层 outline，只保留外层聚焦样式。

## 图片素材管理

图片素材支持：

- 修改标签。
- 修改图片标题。
- 通过标题区分相似图片。
- 标签下拉不再被后续卡片遮挡。

## 关键词和图片库级标签

关键词库和图片库不再使用库级标签。

原因是单个关键词和单张图片已有标签字段，库级标签容易造成“整库隐性继承标签”的误解。

标题库仍保留库级标签，因为标题素材本身没有独立标签字段。

## 删除后的列表状态

关键词库、标题库、图片库和知识库列表中删除某个库后，会保留当前页面的 Collection / 筛选 query，并返回素材列表区域。

这样在某个 Collection 下整理素材时，删除后不会跳回全部素材，也不会回到页面最顶部。

## 治理检查

素材首页的治理检查可以发现：

- 未分配 Collection 的素材。
- 未关联 Entity 的知识库。
- 没有关联素材的 Entity。
- 未关联 Entity 的 Case。
- 异常标签分组。
- 重复标签。
- 停用或归档知识库。
- 未向量化知识片段。

建议在批量生成文章前先处理治理检查中的高风险项。
