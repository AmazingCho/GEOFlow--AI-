# 轻量 CRM 与报价使用说明

本文档说明 GEOFlow 新增的轻量 CRM 工作流。该模块不是完整 ERP，不处理财务总账、库存核算、采购或生产排程；它用于把客户、询盘、知识资料和报价初稿连接起来。

## 入口

后台顶部菜单新增 `CRM`。

CRM 下目前包含：

- 客户
- 询盘
- 报价
- 订单
- 售后
- 内容候选

## 客户管理

客户是 CRM 的基础对象。每个客户可关联一个 Collection（业务容器），后续询盘和报价会沿用该容器。

### 必填字段

- **客户名称（联系人）**：联系人的姓名，必填。在列表和详情页中作为主标识显示。

### 选填字段

- **公司名**：客户所在的公司名称，补充信息。在单据打印中会进入买方信息的 Company 栏。
- 业务容器 Collection
- 客户类型
- 行业
- 国家
- 地址
- 来源渠道
- 联系电话
- 邮箱
- 职位
- 官网
- 负责人（内部员工）
- 状态（活跃 / 潜在 / 不活跃 / 暂停合作）
- 备注

客户类型和来源渠道使用受控选项，减少后续筛选和统计时的命名混乱。国家字段提供可搜索候选，仍保留历史值兼容。

### 详情页功能

客户详情页可维护：

- 基础信息（含公司名和联系人划分）
- 跟进记录（可直接在详情页添加）
- 该客户相关询盘列表
- 该客户相关报价列表

### 删除客户

删除客户时系统会提示：将同时删除其关联的询盘、报价、订单和售后工单。确认后执行级联删除。

## 询盘管理

询盘用于记录客户需求，并把需求连接到现有素材系统。

询盘可关联：

- Collection
- 客户
- 负责人（内部员工）
- Entity
- 知识库
- Case
- 标签

选择 Collection 后，Entity、知识库和 Case 推荐选项会限制在当前 Collection 内。

### 详情页功能

询盘详情页展示：

- 需求识别结果（客户、负责人、语言、需求摘要、产品兴趣、回复要点）
- 原始询盘内容
- 关联的 Entity、知识库、Case
- **报价记录**：该询盘生成的所有报价，点击可跳转
- **订单记录**：该询盘关联的所有订单，点击可跳转
- 标签与备注

## AI 需求识别

询盘创建/编辑页提供 `AI 需求识别` 区域。

使用方式：

1. 粘贴客户邮件、聊天记录或询盘正文。
2. 可选择 AI 模型，也可以保留自动选择。
3. 点击分析。
4. 系统会填入需求摘要、产品兴趣、回复要点、缺失问题，并推荐已有 Entity、知识库和 Case。
5. 管理员可以手动增删推荐结果后保存。

注意：

- AI 不会直接创建新 Entity、知识库、Case 或标签。
- AI 只推荐当前已有对象。
- 没有可用 AI 模型时，系统会使用本地关键词匹配作为降级推荐。

## 单据制作（报价管理）

单据（Quotes）是报价和文档生成的核心入口，覆盖从报价到出货的完整单据链。

### 创建方式

- 从客户直接创建新单据
- 从询盘创建：自动带入客户、负责人、Collection；若询盘关联了 Entity，明细会预填 Entity 名称
- 从单据列表页的"新增单据"按钮创建

### 基础信息

- **客户**：必选，选择后可使用"从客户资料带入"自动填充买方信息
- **业务容器**：可选，限制后续 Entity 和图库候选范围
- **负责人**：内部员工
- **单据号**：自动生成，可手动修改
- **文档类型**：报价单 / 形式发票 / 正式发票 / 装箱单 / 合同
- **标题**：必填
- **币种**：默认 USD

### 买方信息

买方信息模块位于基础信息下方，所有字段支持手动编辑。右上角"从客户资料带入"按钮可一键将客户资料填入对应字段：

| 表单标签 | 字段名 | 自动带入来源 |
|---------|--------|-------------|
| 联系人 | `buyer_contact` | 客户的"客户名称（联系人）" |
| 公司名 | `buyer_company` | 客户的"公司名" |
| 电话 | `buyer_phone` | 客户的联系电话 |
| 邮箱 | `buyer_email` | 客户的邮箱 |
| 国家 | `buyer_country` | 客户的国家 |
| 地址 | `buyer_address` | 客户的地址 |

买方信息在打印单据的 Buyer/Importer（或 Consignee/Buyer）面板中展示。联系人优先显示，公司名作为补充（格式：`联系人 · 公司名`）。

### 商业信息

- **Trade Term**：FOB / CIF / EXW 等贸易条款
- **Origin**：原产国
- **装运港 / 目的港**：物流信息
- **运输方式**：Sea Freight / Air Freight 等
- **唛头（Shipping Mark）**：可选，用于装箱单
- **有效期**：报价有效期
- **交期**：预计交付时间
- **质保条款**
- **安装条款**
- **包装条款（Packing）**：默认值 "Standard export wooden case"，可修改

### Bank Account JSON（卖方银行信息）

仅在形式发票打印时展示。首次为空时预填模板 JSON，可手动编辑。标准字段：

- Beneficiary（受益人）
- Bank Name（银行名称）
- Account No.（账号）
- SWIFT
- Currency（币种）
- Bank Address（银行地址）

### Seller Company JSON（卖方公司信息）

用于覆盖打印页的卖方信息。首次为空时预填模板 JSON。字段：

- name（公司名）
- address（地址）
- phone（电话）
- email（邮箱）
- website（网站）

打印时优先级：单据级 `Seller Company JSON` > 站点设置默认值。

### 明细行

每条明细包含：

- **行类型**：product / accessory / service / spare_part / training / other
- **Entity 关联**：选择已有 Entity 可自动预填型号和描述
- **型号**：产品型号
- **HS Code**：海关编码
- **产品图片**：从图片库选择或本地上传（单张不超过 200KB）
- **名称 / 描述**
- **数量 / 单位**
- **单价**
- **包装件数 / 净重 / 毛重 / 体积**：物流字段，报价单和形式发票阶段可留空
- **包装尺寸 L×W×H (cm)**：装箱单专用

明细小计 = 数量 × 单价，空白行不保存。支持动态增删行。

### 单据类型联动

选择不同单据类型时，明细中的部分字段自动显示/隐藏：

- **报价单 / 形式发票**：显示价格、有效期、银行信息（形式发票）；物流字段可折叠
- **正式发票**：显示价格和 HS Code，顶部含 Exporter/Seller + Importer/Buyer 双面板
- **装箱单**：隐藏价格，显示包装尺寸、重量、体积，顶部含 Shipper/Seller + Consignee/Buyer 双面板，含 Shipment / Shipping Mark / Port Info 三面板
- **合同**：显示价格和自定义合同条款，右侧含 Seller Info 面板

### 金额计算

- 明细小计 = Σ(数量 × 单价)
- 最终合计 = 明细小计 + 运费 + 税费 - 折扣

### 打印页

打印页按文档类型分流渲染，各类单据标题和面板命名如下：

| 单据类型 | 卖方面板 | 买方面板 |
|---------|---------|---------|
| 报价单（Quotation） | - | Buyer / Customer |
| 形式发票（PI） | - | Buyer / Customer |
| 正式发票（CI） | Exporter / Seller | Importer / Buyer |
| 装箱单（PL） | Shipper / Seller | Consignee / Buyer |
| 合同（Contract） | Seller Info | Buyer/Importer |

所有打印页卖方联系人（Contact）映射到单据负责人（owner）。打印页底部签名区 Seller 方的 Name 同样取自负责人。

每条单据的打印页可通过单据列表的"打印"下拉选择目标类型，也可在详情页通过"打印单据..."下拉选择。

## 订单管理

订单用于轻量跟踪报价被接受后的执行状态。

### 创建方式

- 单据详情页点击"生成订单"
- 单据列表页操作列点击"创建订单"

系统会复制报价的客户、Collection、明细、币种和总金额。在订单详情或编辑页维护付款、生产、交付和订单状态。

订单管理只做状态跟踪，不处理库存、财务、采购或生产排程。

## 售后工单

售后工单用于记录真实问题，并把问题连接到现有知识体系。

工单可关联：

- Collection
- 客户
- 订单
- 核心 Entity
- 多个知识库
- 多个 Case

工单创建/编辑页提供 `AI 工单分析`，使用方式与询盘 AI 需求识别类似。

## 内容候选草稿

内容候选用于把 CRM 中的真实需求转成可入库素材，但不会直接写入素材库。

- 从询盘生成标题候选 / FAQ 候选
- 从售后工单生成 FAQ 候选 / Case 候选
- 在 `CRM > 内容候选` 中审核，确认应用后写入对应素材库
- 拒绝的候选不会写入任何素材库

## 创建任务时引用 CRM 来源

创建任务页 `CRM 来源` 区域可选来源：客户、询盘、售后工单。用于记录文章来源、方便后续回溯。

## 推荐工作流

1. 创建或选择 Collection
2. 创建客户，填写联系人姓名和公司名
3. 创建询盘，选择客户、Collection 和负责人
4. 使用 AI 需求识别，关联 Entity、知识库和 Case
5. 从询盘生成单据，选择文档类型，完善买方信息、商业信息和明细
6. 通过打印页预览，打印或发送给客户
7. 报价被接受后，从单据生成订单
8. 订单执行或售后过程中，创建售后工单
9. 将高频问题沉淀为内容候选，人工确认后写入知识库或 Case DB
10. 创建文章任务时，可选择相关询盘、客户或售后工单作为 CRM 来源
