# GEOFlow 新增功能使用总览

本文档汇总从本轮定制开发以来新增的主要功能，以及它们在实际内容生产中的职责边界。

## 核心思路

GEOFlow 现在不再只是“选择素材生成文章”，而是升级为：

1. 先用 Collection 划分业务边界。
2. 再用 Entity 建立知识索引节点。
3. 用知识库保存真实资料。
4. 用 Case 保存真实案例。
5. 用标签描述可复用属性。
6. 创建任务时通过 Collection、Entity、Case、标签和知识库共同限定生成上下文。
7. 文章生成后自动记录来源、RAG 引用和质量评分，方便审核。

## 新增模块职责

| 模块 | 作用 | 不建议承担的职责 |
| --- | --- | --- |
| Collection | 顶层业务容器，例如自动化设备、工业制冷、色选、照明 | 不要用普通标签替代 Collection |
| Entity | 知识索引节点，例如 SJ4060、桌面点胶机、PU 树脂、电池制造 | 不要把所有长文资料塞进 Entity 字段 |
| Knowledge Base | 保存产品手册、FAQ、竞品资料、故障排查等真实资料 | 不再存放案例型内容 |
| Case DB | 保存客户案例、应用场景、项目结果 | 不作为普通知识库 role |
| Tag | 描述轻量可复用属性，例如 Topic、Audience、Intent | 不创建无限分组，不承担 Collection、Entity 或知识库资料类型职责 |
| Article Quality | 判断文章是否完整、是否引用知识库、是否覆盖关键词、是否存在审核风险 | 不自动替代人工最终审核 |

## 推荐使用顺序

1. 创建 Collection。
2. 创建或导入 Entity。
3. 导入知识库，并选择“资料用途”、Collection 和关联 Entity。
4. 创建 Case，并关联最直接的 Entity。
5. 给关键词、图片、知识库、Entity、Case 选择受控标签。
6. 在创建任务页选择 Collection。
7. 根据文章主题选择相关 Entity 和 Case。
8. 按需使用受控分组标签进一步筛选素材。
9. 生成文章后查看“生成来源”和“质量检查”。
10. 对低分文章优先人工审核。

## 关键变化

- Collection 是必选业务容器，标签不再承担顶层分类职责。
- Entity 不是资料正文仓库，而是把知识库、图片、关键词、Case 串起来的索引节点。
- 知识库可以关联多个 Entity，并且每个 Entity 都可以单独设置关系，例如主资料、辅助参考、应用参考或竞品参考。
- Case 类型已经改为受控选项，生成时会根据案例类型决定引用方式。
- 知识库 role 会影响 RAG 检索和生成时的使用方式。
- URL 智能采集生成的 Case 会归一到受控 Case 类型，避免继续产生自由类型。
- URL 智能采集创建任务时可以选择 AI 分析模型；不指定时按模型优先级自动选择。
- Knowledge、Entity、Case 的 AI 自动分析已统一增强提示词规则，重点约束语言一致、事实准确和表格参数保真。
- AI 自动分析区提供“补充分析要求”，用于临时强调表格、语言或案例提取策略，但不会覆盖系统字段规则。
- 标题库可以保留库级标签；关键词库和图片库不再使用库级标签，避免误以为库内素材都继承该标签。
- 标签分组已经改为白名单治理，可在标签管理页维护允许使用的分组。
- 关键词库、标题库、图片库和知识库删除后，会保留当前 Collection / 筛选视图并回到列表位置。
