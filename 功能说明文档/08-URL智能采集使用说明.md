# URL 智能采集使用说明

URL 智能采集用于从目标页面提取素材，并生成可入库的知识库、关键词、标题、Entity、Case 和图片素材。

## 内容语言

内容语言支持：

- 指定中文。
- 指定英文。
- 自动检测。

选择自动检测时，系统会根据目标 URL 页面语言推断生成语言。例如页面是西班牙语，则 AI 知识库、关键词库和标题库应使用西班牙语。

URL 采集生成 Entity 描述时也会遵循采集语言。

例如英文页面不会再套用中文固定模板，避免出现中英文混杂描述。

## AI 分析模型

创建 URL 采集任务时可以选择 AI 分析模型。

可选方式：

- 自动选择模型：默认选项，系统按 AI 模型优先级选择可用聊天模型。
- 指定某个模型：系统会优先用这个模型完成页面清洗、知识库整理、关键词生成和标题生成。

指定模型失败时，系统仍保留原有 failover 机制，会继续尝试其他可用模型。

建议：

- 采集结构复杂、表格较多的资料时，选择理解能力更强的模型。
- 大量普通页面批量采集时，可以使用成本更低的模型。
- 如果某个模型经常导致语言混杂或 JSON 输出不稳定，可以在采集任务中临时切换模型测试。

## 素材生成选项

采集配置中支持选择生成：

- 知识库
- 关键词
- 标题
- 图片
- Entity
- Case

Entity 和 Case 也有独立勾选项，便于控制是否入库。

URL 采集生成 Case 时，会使用受控 Case 类型：

- 页面中存在明确客户结果时，优先整理为客户成功案例或 ROI/指标案例。
- 页面主要描述场景和用法时，归入应用场景案例。
- 页面主要描述故障、问题或解决步骤时，归入问题排查案例。
- 无法明确判断时，归入通用案例。

系统不再保留 `URL采集案例` 这类自由类型，避免 Case DB 后续筛选和生成引用规则混乱。

URL 采集生成 Entity 时，Entity 仍只是索引节点：

- Entity 描述应保持简短。
- 具体产品参数、FAQ、表格、应用边界应进入知识库。
- 真实应用故事、客户结果和指标应进入 Case。

URL 采集生成标题库时，会尽量把每条标题关联到本次生成的关键词：

- 如果标题中包含某个关键词，优先关联该关键词。
- 如果标题和关键词数量不一致，则按关键词列表顺序轮换兜底。
- 标题库会记录关联的关键词库，单条标题也可以在标题库详情页手动编辑关联关键词。

## 采集结果确认

采集完成后，确认内容模块会按素材类型展示候选素材。

每一种素材类型内，每一个素材都有独立勾选框。

默认全部选中。

可以取消勾选不想保存的素材。

每个素材类型卡片右上角有全选入口。

## 推荐使用方式

1. 先采集官方产品页或产品手册页。
2. 检查自动生成的 Entity 是否准确。
3. 保留产品手册、FAQ、参数等知识库素材。
4. 如果页面中没有真实客户案例，不建议强行保存 Case。
5. 入库后进入知识库或 Entity 页面补充 Collection、资料用途、标签和关联关系。
6. 如果一份知识库同时解释多个对象，在知识库详情页为每个 Entity 单独设置关系。
7. 如果采集页包含大量表格，入库前重点检查 Markdown 表格是否保留原始行列、单位和数值。

## 注意事项

- URL 采集生成的内容只是初步素材，不等于完成治理。
- 采集后的知识库需要设置 role，否则 RAG 使用时可能无法体现优先级。
- 采集出的 Entity 信息通常较短，后续仍要通过关联知识库和 Case 补充上下文。
- 如果标题是英文或其他语言，后续文章生成会优先按标题和关键词判断目标语言，并强制正文使用对应语言。
