# 文章编辑器与模板复刻使用说明

本文档说明 2026-06-12 融入的上游能力：文章 Markdown 编辑器、公众号/微信格式复制、编辑器图片上传，以及网站设置中的模板工厂。

## 一、文章 Markdown 编辑器

入口：

- 后台文章创建页
- 后台文章编辑页

主要能力：

- 使用 Markdown 编辑正文。
- 支持标题、列表、表格、引用、代码块等常见 Markdown 结构。
- 支持快捷插入：
  - 图片
  - 二级标题
  - 引用
  - 列表
  - 分隔线
- 支持右键打开常用排版菜单。
- 支持一键复制 Markdown 正文。
- 支持复制公众号/微信格式 HTML。

## 二、复制公众号/微信格式

用途：

- 将文章正文复制到公众号、微信富文本编辑器或其他富文本编辑器中。
- 保留标题、段落、表格、引用、图片等基础排版。

注意：

- 系统不会把文章直接发布到公众号。
- 复制动作只把当前文章正文转换为带内联样式的 HTML。
- 导出时会剥离不安全 HTML，例如 `<script>`。
- 最终在外部编辑器中的显示效果仍受目标平台规则影响，粘贴后建议人工预览。

## 三、编辑器图片上传

入口：

- 文章编辑页的图片快捷插入。

规则：

- 文章必须先保存，进入编辑页后才能上传图片。
- 支持 JPG、JPEG、PNG、GIF、WebP。
- 单张图片最大 10MB。
- 上传后系统会：
  - 创建或复用名为“文章编辑器图片”的图片库。
  - 在图片库中新增图片记录。
  - 建立文章与图片的关联。
  - 自动插入 Markdown 图片语法。

建议：

- 用清晰的图片标题或 alt 文本区分相似图片。
- 编辑器上传适合文章内临时补图；批量产品图、案例图仍建议先进入图片库统一维护标签、Entity 和 Collection 关系。

## 四、模板工厂 / 模板复刻

入口：

- 后台“网站设置”
- “网站模板”区域
- “模板工厂”卡片

当前本地 Docker 可访问入口示例：

```text
http://localhost:18080/admin/site-settings
```

如果使用自定义后台路径，请以实际后台入口为准。

## 五、模板复刻任务如何创建

创建任务需要填写：

- 模板名称
- 主题目录 ID
- AI 分析模型
- 可选基础主题
- 复刻方向
  - 内容站优先
  - 品牌站优先
  - 新闻站优先
- 首页参考 URL
- 列表页参考 URL
- 文章页参考 URL
- 合规确认

URL 安全规则：

- 只允许公开可访问的 `http` / `https` 页面。
- 不允许 localhost。
- 不允许私有 IP、保留地址或内网地址。
- 不建议使用需要登录、频繁跳转或反爬严重的页面。

## 六、模板复刻任务生命周期

任务状态大致包括：

- queued：等待队列执行
- fetching：抓取参考页面
- extracting：提取结构和样式特征
- analyzing：AI 分析
- generating：生成主题草稿
- scanning：安全扫描
- ready：等待人工审核
- published：已发布
- archived：已归档
- failed：失败

管理员可进行的操作：

- 重新入队
- 打开桌面/移动预览
- 提交反馈生成下一版草稿
- 复制为新模板
- 发布模板
- 下载主题包
- 归档任务
- 删除草稿文件

## 七、重要边界

模板复刻不是“直接复制目标网站”。

系统设计目标是：

- 学习布局、模块结构、排版节奏和视觉特征。
- 生成符合 GEOFlow 前台主题契约的 Blade 模板。
- 保留人工审核和发布确认。

系统不会也不应：

- 直接复制商标、受版权保护内容或目标站素材。
- 未经确认覆盖当前启用主题。
- 从私有网络或登录页面抓取内容。

## 八、发布与回滚建议

发布前建议：

- 先看首页、列表页、文章页预览。
- 检查移动端预览。
- 检查是否有外部脚本、远程图片或不安全 Blade。
- 用反馈迭代生成新版本，而不是手工直接改草稿。

发布后：

- 模板会进入主题目录，可在网站设置中启用。
- 如果环境不能直接写入主题目录，可以下载主题包，人工同步。
- 不满意时可切回默认主题或其他已有主题。

## 九、与现有主题系统的关系

- 模板工厂是新增能力，不替代已有主题选择。
- 已有主题目录和默认前台仍保留。
- 生成的主题先进入隔离草稿，只有人工确认发布后才进入可选主题目录。
- 分发渠道的模板选择仍按现有渠道配置规则工作。
