# 素材系统推荐使用方法

本文档说明在已经具备 Collection、Entity、Knowledge Base、Case DB、标题库、关键词库和标签系统后，应该如何组织和使用素材。

## 核心结论

现在系统的主线不再是标签，而是：

```text
Collection 定业务范围
Entity 定引用对象
Knowledge Base 提供事实
Case 提供真实案例
Title 决定文章方向
Prompt / Skill 决定文章写法
Tag 只做辅助筛选
```

标签仍然有价值，但不再承担主分类、业务容器或精准引用职责。

## 各模块职责

| 模块 | 当前职责 | 不建议承担的职责 |
| --- | --- | --- |
| Collection | 顶层业务容器，例如自动化设备、工业制冷、照明设备 | 不要用标签替代 Collection |
| Entity | 知识索引节点，例如 SJ4060、视觉灌胶机、PU Resin | 不要把完整资料正文塞进 Entity |
| Knowledge Base | 保存产品手册、FAQ、参数表、竞品分析、故障排查等真实资料 | 不要保存真实客户案例 |
| Case DB | 保存真实应用故事、客户案例、项目效果和指标 | 不要当作普通知识库使用 |
| 标题库 | 控制文章方向和选题 | 不承担事实来源职责 |
| 关键词库 | 支撑 SEO/GEO 词库、标题生成、关键词覆盖检查 | 不作为主要 RAG 来源 |
| Tag | 描述轻量属性，例如 Topic、Audience、Intent | 不承担 Collection、Entity、知识库类型职责 |

## 推荐建库顺序

1. 创建 Collection。
2. 创建核心 Entity。
3. 导入或创建知识库。
4. 将知识库关联到 Entity，并设置关系。
5. 创建 Case，并关联最核心的 Entity。
6. 创建或导入标题库。
7. 维护关键词库。
8. 只在必要时补充标签。

## Collection 如何使用

Collection 是最顶层业务边界。

适合创建为 Collection 的对象：

- 自动化设备
- 工业制冷
- 色选设备
- 照明设备

原则：

- 不同行业或业务线优先用 Collection 分开。
- 不要用标签分组代替 Collection。
- 创建任务时必须先选择 Collection。

## Entity 如何使用

Entity 是系统识别和引用素材的核心节点。

适合创建为 Entity 的对象：

- 产品型号：SJ4060、DJ771
- 产品线：视觉灌胶机、桌面点胶机
- 材料：PU Resin、Epoxy
- 应用场景：Battery Manufacturing、Wafer Processing
- 技术/工艺：Vision Dispensing、Vacuum Potting

Entity 本身不需要写很长。

Entity 只需要帮助系统知道：

- 这个对象叫什么。
- 属于什么类型。
- 有哪些别名。
- 和哪些知识库、图片、关键词、Case 有关系。

真正丰富的信息应放在知识库和 Case 中。

## 知识库如何使用

知识库是 RAG 检索的主要事实来源。

适合放入知识库：

- 产品手册
- FAQ
- 参数表
- 安装说明
- 故障排查
- 竞品分析
- 技术说明
- 应用边界

知识库创建后，建议做三件事：

1. 选择 Collection。
2. 选择资料用途。
3. 关联 Entity，并设置关系。

常见关系：

- 主资料 / 主体知识
- 辅助参考
- 应用参考
- 竞品参考
- 故障/问题排查参考

## Case 如何使用

Case 用于保存真实应用故事。

适合放入 Case：

- 某客户使用某设备解决了具体问题。
- 某行业场景中的应用过程。
- 某项目的结果、指标或效果。
- 某故障排查过程。
- 某选型或对比验证过程。

Case 目前建议关联一个最核心 Entity。

如果一个案例涉及多个对象，建议：

- Case 关联最核心的产品型号或应用场景。
- 其他背景信息写入 Case 内容。
- 通过知识库多 Entity 关系补充上下文。

## 标题库如何使用

标题库决定文章写作方向。

例如：

- How to Choose a Vision Dispensing Machine
- SJ4060 vs Manual Dispensing
- Best Glue Dispensing Machine for Battery Manufacturing

创建任务时，标题库仍然需要手动选择。

标题库可以保留库级标签，因为标题本身通常不单独维护标签。

## 关键词库如何使用

关键词库现在不是主要 RAG 来源。

它更适合用于：

- 生成标题。
- 管理 SEO/GEO 方向。
- 检查文章关键词覆盖。
- 辅助质量评分。
- 帮助理解某个 Collection 或 Entity 的搜索需求。

关键词库不建议承担事实来源职责。

## 标签系统如何使用

标签系统现在只做辅助筛选。

推荐保留少量受控分组：

- Topic
- Audience
- Intent

示例：

- Topic: troubleshooting
- Topic: product-selection
- Audience: engineer
- Audience: buyer
- Intent: comparison
- Intent: buying-guide
- Intent: application

创建任务页现在也会根据标题库名称和标题样本智能推荐 Skill Prompt。Intent 标签仍然可以作为素材筛选条件，但不再是触发 Skill 的唯一方式；如果标题库已经清楚表达 comparison、buying guide 或 application，系统会给出推荐，用户可手动覆盖。

适合使用标签的情况：

- 同一个 Collection 下素材很多，需要进一步筛选。
- 同一个 Entity 下资料很多，需要按主题或意图缩小范围。
- 创建任务时想进一步限定文章类型。
- 做质量检查或统计时需要轻量属性。

不适合用标签表达：

- 行业大类，应使用 Collection。
- 产品型号，应使用 Entity。
- 产品线，应使用 Entity。
- 知识库资料类型，应使用资料用途。
- 案例类型，应使用 Case 类型。

## 创建任务推荐流程

1. 选择 Collection。
2. 选择标题库或标题。
3. 选择 Master Prompt。
4. 按需选择 Skill Prompt。
5. 选择核心 Entity。
6. 按需选择 Case。
7. 知识库通常让系统通过 Entity 自动检索。
8. 标签筛选默认可以不选。
9. 图片配置按是否需要配图选择。
10. 生成文章。
11. 在文章详情页检查生成来源、质量评分和内链建议。

## 示例：SJ4060 产品文章

假设要生成一篇关于 SJ4060 的英文选型文章。

推荐配置：

| 项目 | 示例 |
| --- | --- |
| Collection | 自动化设备 |
| Entity | SJ4060 |
| 知识库 | SJ4060 产品手册、SJ4060 FAQ、SJ4060 参数表 |
| Case | SJ4060 在电池制造中的应用案例 |
| 标题 | How to Choose a Vision Dispensing Machine for Battery Manufacturing |
| Skill Prompt | Buying Guide 或 Application |
| 标签 | 可不选，或选择 Intent: buying-guide |

这时系统会优先通过 SJ4060 关联资料找到事实来源，而不是依赖标签猜测。

## 最小维护原则

为了避免素材系统越来越乱，建议遵守：

- 能用 Collection 表达的，不用标签表达。
- 能用 Entity 表达的，不用标签表达。
- 能用知识库资料用途表达的，不用标签表达。
- 能用 Case 类型表达的，不用标签表达。
- 标签只保留轻量、可复用、跨素材的辅助属性。

## 快速判断表

| 你想表达的是 | 应该使用 |
| --- | --- |
| 这是哪个业务线 | Collection |
| 这是哪个产品/型号/产品线/场景 | Entity |
| 这是产品手册/FAQ/参数表 | Knowledge Base 资料用途 |
| 这是客户案例/应用场景/ROI | Case DB + Case 类型 |
| 这篇文章要写什么 | 标题库 |
| 这篇文章用什么写法 | Master Prompt + Skill Prompt |
| 这是比较/选型/排障/工程师受众 | Tag |
