# 创建任务与生成流程使用说明

创建任务页已经重新整理为以 Collection 为入口的生成流程。

## 基本流程

1. 选择 Collection。
2. 填写任务基础信息。
3. 选择标题库、Master Prompt、可选 Skill Prompt 和模型。
4. 选择 Entity 和 Case。
5. 按需展开受控分组标签进行筛选。
6. 配置知识库、图片和发布方式。
7. 创建任务并生成文章。
8. 在文章详情页查看生成来源和质量检查。

## Collection 必选

Collection 是创建任务的必选项。

它用于限定本次任务属于哪个业务容器，避免生成流程跨行业、跨业务线误用素材。

选择 Collection 后，Entity 和 Case 选择器会自动限制为当前 Collection 下的内容。

如果确实需要跨 Collection 引用，需勾选“允许跨 Collection 引用”。否则，后端提交时也会拒绝不属于当前 Collection 的 Entity 或 Case。

## Entity 和 Case 选择

Entity 和 Case 支持多选 chip 样式。

推荐原则：

- 选择 1 到 3 个核心 Entity。
- 只选择与文章主题强相关的 Case。
- 不要一次选择过多 Entity，避免上下文发散。

## 受控分组标签

受控分组标签默认以折叠区域展示。

它是可选辅助筛选项，适合进一步限定 Topic、Audience、Intent 等轻量属性。

如果已通过 Collection 和 Entity 精准限定素材，可以不选择受控分组标签。

## Master Prompt 和 Skill Prompt

内容配置区包含两个提示词入口：

- 内容提示词：必选，也就是 Master Prompt，决定正文生成的基础规则、语气、结构和变量使用方式。
- Skill Prompt：可选，用于叠加某类文章结构策略，例如对比文、选型指南、应用场景或排障文章。

不选择 Skill Prompt 时，生成流程保持原来的 Master Prompt 单独生成方式。

选择 Skill Prompt 后，系统会先组合 Master Prompt 与 Skill Prompt，再注入标题、关键词和 RAG 检索出的 `{{Knowledge}}`。

注意：

Skill Prompt 不承担 Collection、Entity、Tag 或知识库分类职责，它只影响文章写法和结构。

当前 Skill Prompt 需要在任务创建页人工选择。

系统还没有实现“根据标题自动识别意图并自动匹配 Skill”的功能。后续如果要做自动匹配，应保留人工覆盖入口。

## AI 模型选择

创建文章任务时的 AI 模型用于正文生成。

它和 URL 智能采集任务中的 AI 分析模型不是同一个入口：

- URL 采集模型：用于网页清洗、知识库整理、关键词和标题生成。
- 文章任务模型：用于根据标题、关键词、RAG 上下文和提示词生成正文。

如果 URL 采集结果质量不好，可以在 URL 采集创建页切换采集模型重新测试。

如果文章正文质量不好，应在创建任务页切换文章生成模型，或调整 Master Prompt / Skill Prompt。

## 图片配置

图片配置区支持：

- 不指定图库：默认选项，系统仍可从可用图片中自动匹配。
- 选择关联图库：优先选择与所选 Entity 相关的图库或图片。
- 不配图：在配图数量中选择，表示明确关闭图片。
- 自动匹配：系统根据文章结构和素材规则选择合适数量。
- 固定数量：按指定数量配图。

注意：

“不指定图库”和“不配图”不是一回事。

- 不指定图库：仍允许系统配图。
- 不配图：完全不使用图片。

## 发布配置

文章列表中新增发布入口后，可对已生成文章单独发起发布。

发布时会弹出确认框，显示可选发布渠道。确认后文章进入发布队列并执行发布流程。

## 生成后的检查

生成完成后建议检查：

- 生成来源是否包含预期 Entity、Case 和知识库。
- 质量评分是否偏低。
- 是否缺少 FAQ、图片或关键词覆盖。
- 是否使用了错误 Collection 或错误素材。
