# Entity 与 Case 使用说明

Entity 和 Case 是 GEOFlow 结构化素材体系的核心。

## Entity 是什么

Entity 是知识索引节点，用来表示系统需要长期识别和引用的对象。

常见 Entity 类型：

- 产品型号：SJ4060、DJ771
- 产品线：视觉灌胶机、桌面点胶机
- 材料：PU 树脂
- 工艺：晶圆加工、电池制造
- 行业场景：新能源制造、半导体封装
- 品牌或竞品：Color-Dec

## Entity 不是什么

Entity 不是完整资料库。

Entity 的字段包括名称、类型、别名、描述、结构化属性 JSON、来源 URL 和标签。这些字段主要用于识别和索引，不负责承载完整产品手册、FAQ 或竞品分析。

真正丰富的上下文应来自：

- 关联知识库
- 关联 Case
- 关联关键词
- 关联图片

## Case 是什么

Case 用来保存真实客户案例、应用场景和项目结果。Case 类型已经改为受控选项，避免后续出现大量自由命名的案例类型。

适合放入 Case 的内容：

- 某客户使用 SJ4060 解决灌胶效率问题。
- 某工厂使用冷水机降低设备温度。
- 某行业应用中产生的实际效果、数据、痛点和解决方案。

## Case 类型如何选择

当前 Case 类型用于指导文章生成时如何引用案例：

| Case 类型 | 适用内容 | 生成时引用方式 |
| --- | --- | --- |
| 客户成功案例 | 客户背景、方案、结果、可公开成效 | 用作真实客户结果证明 |
| 应用场景案例 | 某行业、工艺或场景下的使用方式 | 用作场景适配和方案价值说明 |
| 问题排查案例 | 故障现象、原因、处理步骤 | 用作问题解释和解决路径 |
| 对比验证案例 | 对比测试、选型验证、差异结论 | 用作对比文章和选型建议证据 |
| 实施交付案例 | 项目落地、交付流程、上线注意事项 | 用作落地过程和协作说明 |
| ROI/指标案例 | 成本、效率、良率、回收周期等数据 | 用作量化收益证据 |
| 通用案例 | 信息较泛或暂未归类的案例 | 只作为辅助事实来源 |

## Entity 与 Case 的关系

Case 目前关联一个最直接的 Entity。

例如：

- Case：新能源电池模组灌胶项目
- 关联 Entity：SJ4060

这样生成文章时，系统可以通过 Entity 找到相关 Case，让内容更像真实业务经验，而不是只写泛泛介绍。

如果一个案例同时涉及产品、材料和应用场景，建议：

1. 将案例关联到最核心的 Entity，例如具体产品型号。
2. 在案例摘要、挑战、方案和结果中写清楚其他对象。
3. 通过知识库与多个 Entity 的独立关系来补充材料、产品线或应用场景上下文。

## Entity 与其他素材的关联

当前系统支持在素材侧或 Entity 侧建立关联，建议按以下方式使用：

- 知识库关联 Entity：表示这份资料解释或约束某个对象；同一知识库可关联多个 Entity，并为每个 Entity 单独设置关系。
- 关键词关联 Entity：表示该关键词与某个对象强相关。
- 图片关联 Entity：表示图片可以用于展示某个对象或场景。
- Case 关联 Entity：表示案例主要发生在这个对象或场景上。

知识库与 Entity 的关系选项包括：

- 主资料 / 主体知识
- 辅助参考
- 竞品参考
- 应用参考
- 故障/问题排查参考

## 创建任务时如何选择

创建任务页支持选择多个 Entity 和多个 Case。

推荐流程：

1. 先选择 Collection。
2. 再选择与文章主题直接相关的 Entity。
3. 如果文章需要真实应用故事，再选择 Case。
4. 不要为了一篇文章选择过多 Entity，否则上下文会变散。

## AI 自动识别分析

新增 Entity 和新增 Case 页面已加入 AI 自动识别分析区。

使用方式：

1. 在页面顶部文本框粘贴产品介绍、案例描述或资料片段。
2. 选择 AI 模型。
3. 点击“分析”。
4. 系统会自动识别名称、类型、描述、标签、属性等信息并填入表单。

AI 识别结果建议人工确认后再保存。

## 补充分析要求

Entity 和 Case 的 AI 自动分析区支持展开“补充分析要求”。

这个输入框不是完整替换系统提示词，而是补充本次分析重点。

适合填写：

- 指定输出语言，例如“请用英文输出描述和字段内容”。
- 强调产品型号、别名、参数、应用场景。
- 要求只有存在真实问题、方案、结果或指标时才提取 Case。
- 要求指标必须来自原文，不要编造 ROI 或效率提升。

不建议填写：

- 要求 AI 忽略字段结构。
- 要求把整段资料直接塞入 Entity 描述。
- 要求从普通产品说明中强行生成客户案例。

系统会继续强制执行：

- Entity 只能作为轻量索引节点。
- Entity 类型和 Case 类型归一到受控选项。
- Case 指标不得虚构。
- 输出语言应与材料或指定语言保持一致。
