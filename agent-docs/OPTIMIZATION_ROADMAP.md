# GEOFlow 优化路线图

本文档记录后续 agent 应理解的优化阶段，不依赖聊天上下文。

## 主线阶段

| 阶段 | 名称 | 目标 | 当前状态 |
| --- | --- | --- | --- |
| 1 | 生成追踪基础 | 记录文章生成来源、任务上下文和素材引用 | 已完成 |
| 2 | 正式 RAG 检索 | 让文章生成使用知识库 chunk、Entity、Case 上下文 | 已完成 |
| 3 | 自动标签推荐 | 素材创建或编辑时辅助推荐已有标签 | 已完成核心功能 |
| 4 | Entity / Case 结构化 | 增加结构化素材，支持案例和实体关系 | 已完成核心功能 |
| 5 | GEO 生成流水线 | 按 Collection、Entity、知识库、Case 组织生成流程 | 已完成核心功能 |
| 6 | 质量评分与审核辅助 | 让系统自动判断文章质量并提示问题 | 已完成核心功能 |
| 7 | 性能与后台体验优化 | 大数据量下减少卡顿，优化筛选和列表交互 | 已完成部分，需继续压测 |

## 治理增强阶段

| 阶段 | 名称 | 目标 | 当前状态 |
| --- | --- | --- | --- |
| G1 | Collection 规范 | Collection 作为顶层业务容器，替代行业标签承担容器职责 | 已完成 |
| G2 | Entity 类型规范 | 保持共享 Entity 表，使用类型和 attributes JSON 管理差异 | 已完成规则，仍可加强 UI 说明 |
| G3 | Tag 分组白名单 | 限制可用标签分组，减少标签治理混乱 | 已完成核心功能 |
| G4 | 知识库 role 使用约束 | 明确 primary、supporting、constraint、comparison、style、archive 用法 | 已完成核心功能 |
| G5 | 重复/无效标签治理 | 检查异常分组、重复标签、无效标签 | 已完成部分 |
| G6 | 素材关联完整性检查 | 检查素材是否缺 Collection、Entity、向量化或状态异常 | 已完成部分 |

## 后续独立阶段

### 任务回收站

目标：

- 删除任务后保留文章。
- 文章来源显示“任务已删除”。
- 被删除任务进入回收站。
- 支持恢复或永久删除。

状态：未实现。已有详细优化提示词可继续使用。

### AI 知识库纠错助手

目标：

- 在知识库和文章详情页发起纠错。
- AI 根据错误描述检索相关 chunk。
- 生成 correction proposal。
- 管理员确认后才更新知识库 chunk。
- 更新后重新 embedding。
- 保留版本和回滚能力。

状态：未实现。已有详细优化提示词可继续使用。

## 下一步建议

1. 先继续完善阶段 7 的真实大数据性能验证。
2. 再实现任务回收站，避免误删任务影响文章来源。
3. 再实现 AI 知识库纠错助手，建立知识更新闭环。
4. 后续可增加 Collection 健康度评分和素材重复合并建议。
