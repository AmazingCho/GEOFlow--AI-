# GEOFlow 更新日志

该文档记录公开仓库可见功能的持续更新。后续每次推送到 GitHub 时，同步更新本文件和英文版 `CHANGELOG_en.md`。

## 2026-06-05

### 文章 Skill Prompt v1

- 新增轻量级 Master Prompt + Skill Prompt 生成机制：
  - 保留原 `prompt_id` 作为任务必选 Master Prompt，不改变旧任务正文生成逻辑。
  - 新增 `tasks.skill_prompt_id`，任务创建/编辑页可选 Skill Prompt。
  - Worker 生成文章时会将 Master Prompt 与 Skill Prompt 拼接后再渲染标题、关键词和 RAG 知识变量。
  - 生成追踪中记录 `skill_prompt_id`、`skill_prompt` 与 `has_skill_prompt`，便于文章审核时回溯。
- 扩展“文章提示词配置”入口：
  - 原正文提示词页面同时管理 `content` Master Prompt 与 `skill` Skill Prompt。
  - 新增默认 Skill Prompt 模板：Comparison、Buying Guide、Application。
  - 已被任务引用的提示词不允许直接切换类型，避免旧任务引用语义改变。
- API catalog 增加 `skill_prompts` 输出，方便后续外部任务创建入口复用。
- 验证：
  - 已执行迁移 `2026_06_05_010000_add_skill_prompt_id_to_tasks`。
  - `WorkerExecutionServicePromptTest`、`AdminAiPromptsPageTest`、`AdminTasksPageTest` 和相关 API catalog / task create 测试通过。

### AI 素材分析提示词与 URL 采集语言修复

- URL 智能采集创建任务新增 AI 分析模型选择：
  - 新建采集任务时可选择具体聊天模型，也可保留“自动选择模型”。
  - 指定模型会优先用于 URL 页面清洗、知识库整理、关键词和标题生成。
  - 保留原有失败切换机制，指定模型不可用或调用失败时仍会尝试其他可用模型。
- 新增素材 AI 分析公共规则：
  - 抽出语言一致性、事实不可编造、表格/参数保真和 JSON 输出约束。
  - 手动知识库、Entity 和 Case 的 AI 自动分析统一复用这些规则。
  - 知识库分析更明确地区分摘要、后台描述和可入库 Markdown 正文，避免整段原文简单复制到多个字段。
- 优化知识库、Entity 和 Case 的预制分析规则：
  - 知识库重点保留产品参数、FAQ、步骤、边界条件和表格内容。
  - Entity 明确作为轻量索引节点，不再鼓励塞入整段资料。
  - Case 仅在存在真实场景、问题、方案、结果或指标时提取，指标不得编造。
- 新增“补充分析要求”折叠输入区：
  - 知识库新增/详情页、Entity 表单和 Case 表单均可补充本次 AI 分析重点。
  - 提供“表格参数保真”“英文输出”“谨慎提取案例”快捷模板。
  - 补充要求只作为附加约束，不能覆盖系统字段结构、语言规则和事实约束。
- 修复 URL 智能采集 Entity 描述语言混杂：
  - 英文等非中文页面不再套用中文描述模板。
  - Entity 描述改为依据采集语言保留证据/上下文，避免出现“是从 URL 采集内容中识别出的……”这类中文固定句。
- 验证：
  - 新增英文 URL Entity 描述测试和公共提示词规则测试。
  - 相关 PHP 语法检查通过，素材页面聚焦测试通过。

### 知识库 Entity 关系与 Case 类型治理

- 优化知识库与 Entity 的关联工作流：
  - 知识库创建、编辑和详情页复用 Entity 页“多选 + 每项关系下拉”的交互。
  - 同一知识库可关联多个 Entity，并为每个 Entity 独立设置主资料、辅助参考、应用参考、竞品参考或故障排查参考。
  - 保留全局默认关系字段作为旧数据、批量操作和旧表单提交的兜底兼容。
  - Entity 编辑页关联知识库也改为复用同一通用关系选择组件，减少重复 JS 和 UI 规则。
- 新增通用关系多选组件：
  - 新增 `resources/views/admin/partials/relation-multi-selector.blade.php`。
  - 统一处理“选择多个对象后，每个对象单独设置关系”的前端交互。
- 优化 Case 类型治理：
  - 新增受控 Case 类型体系，覆盖客户成功案例、应用场景案例、问题排查案例、对比验证案例、实施交付案例、ROI/指标案例和通用案例。
  - URL 智能采集和 AI 表单分析生成 Case 时会归一到受控类型，不再继续产生 `URL采集案例` 等自由类型。
  - RAG 和 Worker 生成上下文会根据 Case 类型补充引用规则，帮助文章生成更准确地使用案例证据。
- 优化素材列表删除后的操作体验：
  - 关键词库、标题库、图片库和知识库删除后保留当前筛选 query，并回到素材列表区域。
  - 知识库保存视图新增“清除”入口，可清除视图筛选并保留当前 Collection。
- 文档同步：
  - 更新 `功能说明文档/` 中的总览、Entity/Case、知识库 RAG、素材管理和 URL 智能采集说明。
  - 更新 agent 实现状态，方便后续 agent 快速理解当前工作流变化。

## 2026-06-03

### 分类与标签系统简化

- 将默认白名单标签分组收敛为 `Topic`、`Audience`、`Intent`，旧分组数据继续兼容但不再作为推荐默认。
- 创建任务页按 Collection 过滤 Entity、Case、标题库、知识库和图库选项；后端同步校验，默认禁止提交跨 Collection 的库或素材引用。
- 知识库创建/编辑页将“资料类型、使用角色、重要度”合并为单一“资料用途”选择器，后台仍保存原字段以兼容 RAG 和历史数据。
- 移除知识库“案例素材”保存视图，避免继续用 `Content Type:Case Material` 替代 Case DB。
- 更新标签、架构和创建任务说明文档，明确 Tag 不再承担 Collection、Entity 或知识库资料类型职责。

### 治理增强与任务页体验补充

- 受控标签分组改为可维护白名单：
  - 新增 `controlled_tag_groups` 数据表，标签管理页可新增、重命名和删除允许使用的标签分组。
  - 素材治理检查、任务创建页和标签管理页统一读取数据库白名单，替代硬编码分组。
  - 删除白名单分组不会删除已有标签，仅会让该分组在治理检查中被识别为异常分组，保护历史数据。
- 优化创建任务页：
  - Collection 调整为必选项，作为任务生成的顶层业务范围。
  - Entity 与 Case 选择改为搜索式多选 chip 交互，支持选择多个关联对象。
  - 选择 Collection 后，Entity 与 Case 选择器会限制为当前 Collection 下的内容；跨 Collection 引用需要显式开启。
  - 受控分组标签改为 accordion 展示，作为可选高级筛选，降低页面拥挤感。
  - 图片配置支持根据所选 Entity 自动优先展示关联图库，并区分“不指定图库”和“不配图”。
  - 统一下拉菜单、输入框和标签选择器的边框与聚焦样式，减少双层边框和视觉不一致。
- 调整基础素材库与库级标签规则：
  - 基础素材库入口移除业务容器卡片，Collection 仅保留顶部菜单入口。
  - 基础素材库卡片顺序调整为 Entity DB、标题库、关键词库、图片库、Case DB、作者库。
  - 关键词库和图片库移除库级标签入口，避免误解为库内素材隐式继承标签。
  - 标题库保留库级标签，因为标题素材本身没有独立标签字段。
- 增强素材与 Entity 关联：
  - 素材关联 Entity 的选择器支持多选和取消关联，交互复用标签选择器样式。
  - 知识库、关键词、图片、Case 与 Entity 的关系更适合用于 RAG 检索和生成上下文追踪。
- 优化标签与标题管理：
  - 具体标签创建、重命名和批量移动分组时，只能选择白名单标签分组，避免新增非受控分组。
  - URL 智能采集生成标题库时，会关联本次生成的关键词库，并为单条标题自动匹配关键词。
  - 标题库详情页新增单条标题编辑入口，可手动修改标题和关联关键词。
- 优化文章生成语言控制：
  - 正文生成语言优先由标题和关键词判断，提示词语言只作为兜底。
  - 最终生成提示会强制模型使用目标语言，避免英文标题生成中文正文。
- 文档补充：
  - 新增 `功能说明文档/` 目录，按模块整理 Collection、Entity/Case、标签白名单、知识库治理、任务创建、质量审核、素材关联和 URL 智能采集使用说明。

### 自定义素材治理与生成链路增强

- 新增 Collection 顶层业务容器：
  - 支持在后台管理 Collection，并在关键词库、标题库、图片库、知识库、Entity、Case 等素材列表中按 Collection 归属筛选。
  - 移除“行业领域标签承担顶层容器”的设计，避免标签系统和业务容器职责混淆。
- 增强 Entity / Case 结构化素材体系：
  - Entity 支持与知识库、关键词、图片等素材建立关联，Case 可继续关联 Entity。
  - 新增 Entity / Case 创建页 AI 自动识别分析入口，可将粘贴内容解析并填入表单字段。
- 增强知识库治理能力：
  - 知识库新增类型、使用角色、重要度、摘要、来源 URL、状态等元数据。
  - 知识库列表支持保存视图、Collection、类型、role、重要度、Entity、标签分组、状态等筛选。
  - 新增批量治理操作：分配 Collection、追加标签、设置 role、关联 Entity、归档停用、切换状态。
  - `archive` / `inactive` 知识库默认不参与自动 RAG 检索，手动指定时仍可引用，兼容旧任务。
- 优化 RAG 与生成追踪：
  - 文章生成上下文记录 Collection、Entity、Case、知识库、检索 chunk 与质量检查信息。
  - RAG 检索优先使用 Entity 关联知识库与案例作为真实上下文来源，而不是只依赖 Entity 字段。
- 新增文章质量评分与审核辅助：
  - 后台文章列表显示质量评分。
  - 文章编辑页展示语言、关键词、知识库引用、FAQ、图片匹配、结构完整度等质量问题建议。
  - 修复文章列表页质量评分缓存可能与编辑页实时评分不一致的问题。
- 增强素材标签与库级标签管理：
  - 标签选择器改为可搜索交互，避免大量标签平铺。
  - 关键词库、标题库、图片库编辑页支持库级 tag。
  - 标题库列表补充编辑信息入口，与关键词库管理体验保持一致。
- 新增素材治理检查：
  - 素材首页显示治理检查卡片，识别未分配 Collection、未关联 Entity、未向量化、归档/停用知识库、异常标签分组、重复标签等问题。
- 优化创建任务页图片配置：
  - 图库选择默认显示“不指定图库”。
  - “不使用图片”保留在图片数量选项中，作为关闭配图的明确入口。
- 测试覆盖：
  - 补充 Collection、素材治理、知识库批量治理、RAG 检索、文章质量评分、任务创建、Entity 提取、生成追踪等功能测试。
