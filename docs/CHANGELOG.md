## 2026-06-19

### Codex 业务录入助手 API Phase 2-6

- 新增 AI 录入草稿箱数据层：`ai_intake_drafts` 和 `ai_intake_actions`，用于保存 Codex / AI 生成的待审核业务录入建议。
- 新增 `assistant:write` API scope，以及 `POST /api/v1/assistant/intake-drafts/validate`、`POST /api/v1/assistant/intake-drafts`、`GET /api/v1/assistant/intake-drafts/{id}`。
- 草稿创建接口复用现有统一响应信封、Request-Id 和 `X-Idempotency-Key` 幂等机制；重复提交同一草稿不会重复创建记录。
- 新增后台“AI 录入草稿箱”，超级管理员可查看原始输入、AI 摘要、动作内容、风险、置信度、治理提醒，并执行应用或拒绝。

- 知识库和 Case 动作只会创建 `crm_content_proposals` 内容候选，不会直接覆盖知识库正文或直接写入 Case DB。
- 新增基础治理提醒：缺少 Collection、疑似重复客户、低置信度草稿和低置信度动作；草稿列表支持按状态、风险和来源筛选。
- 新增本地调用脚本 `scripts/codex-intake.mjs`，可通过环境变量中的 API Token 调用 context search、create draft 和 show draft。
- 新增用户说明文档 `功能说明文档/12-Codex业务录入助手API使用说明.md`，并同步更新 Agent 文档、白皮书和实现状态。
- 验证：`AssistantIntakeDraftApiTest` 通过，共 7 tests / 73 assertions；新增 PHP、Blade 文件语法检查通过；容器内 `node --check scripts/codex-intake.mjs` 通过。

### 知识库重复资料与冲突检测（阶段 5）

- 新增知识库治理检查页，可从知识库列表进入，对最近更新的知识库或指定 Collection 进行扫描。
- 检查内容包括：内容完全重复、来源 URL 相同、标题相同、标题/内容相似，以及同来源、同 Entity 或同 Collection 相近资料中的数值型事实冲突。
- 收紧冲突判断：不同产品型号、产品线或产品服务主体之间的 power、weight、voltage 等规格差异不再视为冲突；共享品牌/公司只作为上下文，不再单独触发规格冲突。
- 治理页展示扫描统计、疑似重复组、疑似冲突对、置信度、资料来源、关联 Entity 和详情页跳转。
- 本阶段只生成检查报告，不自动删除、合并、覆盖知识库内容，也不修改现有切片、embedding 或 RAG 流程。
- 验证：新增治理服务、路由、页面、翻译和聚焦测试，覆盖重复/冲突识别与 Collection 筛选。

### AI 知识库纠错助手（阶段 3）

- 新增知识库纠错闭环：知识库详情页和文章编辑页都可以发起“AI 知识纠错助手”。
- AI 只生成纠错提案，不会直接覆盖知识库内容；管理员可在纠错详情页批准、拒绝、应用或回滚。
- 新增纠错记录和知识片段版本记录，支持关联文章、知识库和具体知识片段。
- 应用纠错时会更新目标 `knowledge_chunks.content` 和知识库正文中的对应片段，并刷新该片段 embedding / fallback vector。
- 文章编辑页支持选中文章中的错误段落后发起纠错，便于从生成结果反查知识来源。
- 后台新增 `/admin/knowledge-corrections` 纠错记录页，可按状态、知识库和文章筛选。
- 边界保护：无可用 AI 模型或 AI 返回异常时会生成安全待审核提案；如果原知识片段已变化，应用操作会被阻止，避免覆盖新内容。
- 验证：新增 `AdminKnowledgeCorrectionTest` 通过；文章编辑页、知识库详情页和纠错记录页浏览器检查无控制台错误。

### Skill Prompt 智能推荐（阶段 2）

- 创建任务页的 Skill Prompt 从纯手动选择升级为“智能推荐 / 不使用 / 手动指定”三种路径。
- 默认“智能推荐”会根据标题库名称和标题样本识别文章意图，例如 comparison、buying guide、application，并自动匹配对应 Skill Prompt。
- 页面会展示推荐 Skill、置信度、命中样本和推荐原因；用户仍可手动覆盖，或明确选择“不使用 Skill Prompt”。
- 后端提交时会再次执行同一套推荐逻辑，避免前端提示和实际保存不一致；直接 API/表单提交仍兼容旧的整数 `skill_prompt_id`。
- 验证：任务页打开、手动 Skill、自动推荐 Skill、显式不使用 Skill 的聚焦测试通过。
