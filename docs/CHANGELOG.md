# GEOFlow 更新日志

该文档记录公开仓库可见功能的持续更新。后续每次推送到 GitHub 时，同步更新本文件和英文版 `CHANGELOG_en.md`。

## 2026-06-15

### CRM 销售链路 V2：统一活动时间线

- 新增 `CrmActivityService`，客户、询盘、商机共享同一个活动写入逻辑。
- 商机详情页新增活动时间线和活动记录入口。
- 活动记录支持同时创建下一步待办，并自动保留客户、询盘和商机来源链。
- 待办完成时可填写完成结果，系统会同步生成一条关联该待办的活动记录。
- 活动条目展示关联询盘、商机和来源待办，编辑与软删除能力保持不变。
- 验证：`AdminCrmPagesTest` 22 tests / 202 assertions；`CrmPipelineAuditTest` 2 tests / 12 assertions；Blade 全量编译通过。

### CRM 销售链路 V2：询盘转商机统一编排

- 新增事务型 `OpportunityConversionService`，统一询盘详情和全局来源询盘创建流程。
- 转换时原位补关联未完成待办和未归属单据，不复制记录；已完成待办保持历史状态。
- 询盘详情页在转换前展示受影响待办与单据数量，并提供明确确认提示。
- 从询盘或商机创建待办时自动保留完整的询盘、商机和客户来源链，矛盾组合会被拒绝。
- 转换结果写入管理员活动日志。
- 验证：`AdminCrmPagesTest` 21 tests / 185 assertions；Blade 全量编译通过。

### CRM 销售链路 V2：商机生命周期

- 商机新增/编辑页增加“从询盘创建 / 无来源直接创建”模式和可搜索来源询盘选择器。
- 来源询盘会联动客户与 Collection；跨客户、跨 Collection 或重复活动商机会被拒绝。
- 数据库增加同一询盘仅允许一个活动商机的唯一约束。
- 商机支持安全归档、已归档筛选和恢复，关联待办、活动与单据均保留。
- 归档前展示关联影响摘要；来源冲突时阻止恢复。
- 验证：`AdminCrmPagesTest` 19 tests / 172 assertions；`CrmPipelineAuditTest` 2 tests / 12 assertions；Blade 全量编译通过。

### CRM 销售链路 V2：数据契约与只读审计

- 活动记录新增商机和待办关联字段，一个待办可保留多条历史活动。
- 新增只读命令 `crm:pipeline-audit`，检查孤立/重复商机、未关联待办与单据、关系不一致、归档商机未完成待办及活动关联候选。
- 命令支持 JSON 输出和 `--fail-on-issues`，不会自动修改历史数据或猜测有歧义的关系。
- 当前真实数据审计识别出 16 项待治理关系，审计前后 CRM 数据数量一致。
- 验证：`CrmPipelineAuditTest` 2 tests / 12 assertions；`AdminCrmPagesTest` 16 tests / 146 assertions。

### CRM 活动、联系人与商机下一步优化

- 历史活动记录支持编辑和软删除：
  - 客户详情和询盘详情提供常驻的编辑、删除入口。
  - 编辑继续复用轻量 Markdown 编辑器；单据、订单和售后时间线保持只读。
- 修复客户详情新增联系人时，仅填写姓名或留空可选字段会触发 `500` 的问题：
  - 根因是 Laravel 将空字符串转为 `null`，而联系人表相关字段使用非空约束。
  - Controller 现会在写入前统一规范化可选字符串，并安全同步旧客户联系人兼容字段。
- 明确 CRM 三类对象的职责：
  - 活动记录保存已经发生的沟通事实。
  - CRM 待办保存未来要执行的动作、负责人和截止时间。
  - 商机保存阶段、金额、概率、预计成交日期等销售管道信息，并以首个未完成待办显示“当前下一步”。
- 商机表单移除重复的手动“下一步动作 / 下一步时间”输入；旧数据库字段保留，仅用于历史兼容。
- 修复手动新建商机时，留空“竞争对手”触发数据库非空约束并返回 `500` 的问题；金额为空时同步兜底为 `0`。
- 验证：`AdminCrmPagesTest` 全量通过，共 `16 passed / 146 assertions`。
